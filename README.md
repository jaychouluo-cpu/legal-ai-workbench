# 法务智库 · 企业法务 AI 工作台

> 交付版本 v1.0 —— 由《企事业单位法律 AI 工具箱整合方案》（研究对象 awesome-legal-ai-zh）
> 与《企业法务 AI 工作台 · 设计原型》共同落地的前端工程。

一套面向 **企业法务 / 合规、政府事业单位法制、国企综合法务** 三类岗位的法务 AI 工作台前端。
以「一壳多挂」为形态，把整合方案中的 **底座 → 检索 → 业务能力 → 交付协同** 四层能力，
组织成统一入口下的 9 个工作区，并贯彻 **「数据不出域」「license 白名单」** 两条落地红线。

---

## 一、快速开始

```bash
cd 企业法务AI工作台
npm install        # 安装依赖
npm run dev        # 本地开发（默认 http://localhost:5173）
npm run build      # 生产构建 → dist/index.html（单文件、自包含）
npm run preview    # 预览构建产物
```

> 构建产物 `dist/index.html` 为 **单个自包含 HTML**（约 216 KB，已内联全部 JS/CSS），
> 可直接拷贝到内网、双击在浏览器打开运行，无需任何外部 CDN 或联网资源 ——
> 契合整合方案「内网私有化 · 数据不出域」的部署要求。

## 二、与整合方案的对应关系

设计原型给出了视觉与交互规格，整合方案给出了内容与落地约束。本工程把两者合一：

| 整合方案要点 | 在工作台中的落地 |
| --- | --- |
| **四层架构**（L0 底座 / L1 检索 / L2 业务能力 / L3 交付协同） | 工作台首页「工具箱分层架构」卡片；侧边栏按层归类导航 |
| **落地原则：一壳多挂** | 「AI 法务助手」统一对话入口 + 右侧「已挂载能力」按需启停 |
| **红线一：数据不出域** | 顶栏「数据不出域」标识、脱敏开关、设置 → 数据安全；检索仅传查询要素 |
| **红线二：license 白名单** | 会话仅启用 ✅ 可商用能力；设置 → 数据安全「license 白名单」开关 |
| **国产化 / 信创** | 模型底座默认「智海-录问 70B」内网私有化；可选 LexiLaw / DISC-LawLLM |
| **P0 最小可用集** | 合同审查(ContractGuard)、法规检索(北大法宝/cncases)、企业核查(企查查)、脱敏(presidio)、文格门禁 |
| **人工复核兜底** | 全局页脚免责声明：AI 产出须法律人复核、保留可追溯出处 |

### 9 个工作区与功能实现状态

> 标注「✅ 真实可用（本地）」的模块在浏览器内闭环运行、原文不出域，**无需任何 API**；
> 标注「🔌 需外部服务」的模块已接好真实调用钩子，配置后即真实联网工作，未配置时为演示态。

| 工作区 | 实现状态 | 说明 |
| --- | --- | --- |
| 工作台 | ✅ 真实可用 | 能力总览、统计、最近处理、四层架构（取自整合方案） |
| AI 法务助手 | 🔌 需模型服务 | 配置模型 Endpoint（OpenAI 兼容，如智海-录问 /v1）后**真实调用推理**；未配置为演示态并提示去配置 |
| 合同审查 · 红线 | ✅ 真实可用（本地） | **真实规则引擎**：粘贴/编辑合同→识别违约金过高、任意解除、管辖不明、验收模糊、保密无期限、知产归属不清→公平度评分+红线建议+命中证据 |
| 法规 · 案例检索 | ✅ 真实可用（本地） | **真实全文检索**内置法规/案例语料，关键词加权排序+分类筛选+摘要；在线增强需北大法宝账号 |
| 企业核查 · 尽调 | ✅ 规则真实 / 🔌 数据需 API | 录入风险指标→**本地准入规则真实评估**（一票否决/附条件/准予）；真实工商涉诉数据需企查查 API Key |
| 数据脱敏 · 合规 | ✅ 真实可用（本地） | **真实 PII 引擎**：身份证/手机/银行卡/邮箱/姓名/住址正则识别+掩码，实时输入输出、实体统计、导出 |
| 文书生成 · 文格 | ✅ 真实可用（本地） | **真实模板填充**：表单变量→实时预览→格式门禁实时校验→一键导出 Word(.doc) |
| 期限管家 · 台账 | ✅ 真实可用（本地） | **真实 CRUD + 日期计算**：新增/删除事项、按到期日自动算剩余天数、紧急度着色、localStorage 持久化 |
| 设置 | ✅ 真实可用 | 模型与底座、API 与数据接口（已做密钥审计）、数据安全、通知；配置持久化并驱动各模块 |

### API 密钥审计（对应目标二）

「设置 · API 与数据接口」按是否真正需要凭证分三组，**本地能力已取消密钥栏**：

| 能力 | 是否需要凭证 | 配置项 |
| --- | --- | --- |
| 模型服务（智海-录问 等） | 需 Endpoint（私有化时 API Key 可留空） | Endpoint + 模型名 + 可选 Key，含「测试连接」 |
| 企查查 MCP | ✅ 需 API Key | API Key |
| 天眼查 MCP | ✅ 需 API Key | API Key |
| 北大法宝 MCP | ✅ 需账号 | 账号 |
| presidio 脱敏 / ContractGuard 合同审查 / 文格 / 期限管家 / cncases | ❌ 本地运行，**无需密钥** | 仅显示「已就绪 · 本地」，无输入栏 |

## 三、技术与工程结构

- **技术栈**：React 18 + Vite 5，单文件打包插件 `vite-plugin-singlefile`
- **样式策略**：完整沿用设计原型的内联样式规格，经 `src/lib/style.js` 的 `s()` 将样式字符串
  转为 React 样式对象，保证 **与设计稿 1:1 高保真还原**；交互态（hover/focus）见 `src/styles.css`
- **数据集中**：所有演示数据集中在 `src/data.js`，正式落地时替换为真实接口即可

```
企业法务AI工作台/
├── index.html              # 入口
├── vite.config.js          # 单文件打包配置
├── package.json
├── src/
│   ├── main.jsx            # React 挂载（含 SettingsProvider）
│   ├── App.jsx             # 外壳：侧边栏 + 顶栏 + 视图路由 + 全局免责
│   ├── styles.css          # 全局样式 / 字体降级 / 交互态
│   ├── data.js             # 静态展示数据（方案 + 原型）
│   ├── lib/
│   │   ├── style.js        # 样式字符串 → 对象 工具
│   │   ├── storage.js      # localStorage 持久化
│   │   ├── settings.jsx    # 全局配置上下文（endpoint/密钥），驱动各模块
│   │   └── llm.js          # OpenAI 兼容 Chat 客户端（AI 助手用）
│   ├── engine/             # 真实功能引擎（纯本地、可单测）
│   │   ├── mask.js         # PII 脱敏（presidio 对应）
│   │   ├── contract.js     # 合同审查规则引擎（ContractGuard 对应）
│   │   ├── corpus.js       # 法规/案例语料 + 全文检索
│   │   ├── docgen.js       # 文书模板填充 + Word 导出 + 门禁
│   │   └── entity.js       # 供应商准入规则评估
│   ├── components/         # Sidebar / TopBar / icons
│   └── views/              # 9 个工作区视图
└── dist/index.html         # 构建产物（自包含单文件）
```
